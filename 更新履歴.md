# Changelog

## Ver. 1.0 (v1.0.0) - 2026-04-23

### 正式版リリース
- **タイトルバー + 免責バナーを実装**。タイトル「農薬簡易検索アプリ」(20pt、アプリのトーンに合わせた穏やかな配色) の下に、生産者提供時の JPP-NET 確認を求める免責事項バナー (15pt、黄色警告色) を配置。
- **免責事項**: 本アプリは簡易版であり、生産者への農薬登録情報提供時には必ず [JPP-NET](https://web1.jppn.ne.jp/) で確認することを明示。誤使用に対する製作者の無責任条項を追加。
- **配布用 ZIP**: `browser-pesticide-search_v1.0.0.zip` を UTF-8 EFS フラグ付きで生成 (Windows 10/11 の日本語ファイル名互換)。
- **使い方.html**: エンドユーザ向け操作マニュアルを単一 HTML で作成。毎回データ更新推奨の警告付き。
- **ファイル名日本語化**: `README.md` → `使い方.md`、`CHANGELOG.md` → `更新履歴.md`、`pesticide_search_bundled.html` → `これをクリック.html`。

## v0.2.0 - 2026-04-23

### 追加
- **ワンクリック更新ランチャー**: `データ更新.command` (macOS) /
  `データ更新.bat` (Windows) ダブルクリックで FAMIC+CropLife 最新データを
  自動取得→JSON生成→シングルHTML再ビルド。FAMIC index ページから zip URL を
  スクレイピングするため月ごとのファイル名変更に自動追従。
- **cross-platform `scripts/update_data.py`**: Python stdlib のみ使用
  (urllib, zipfile)。curl/unzip 不要。openpyxl は初回自動インストール。
- **成分名ベース RAC フォールバック**: 登録番号でヒットしない新規登録商品も、
  同じ成分が既存商品で分類されていれば自動で RAC コードを付与。
- **手動 RAC ラベリング** (`rawdata/rac_manual.json`): RAC 体系対象外の
  物理剤/生物製剤/軟体動物剤/動物薬/無機剤や古典剤を手動分類。
  `対象外` / `古典未分類` / `新規未収録` の3ステータスで UI 表示。
- **RAC 分類完全カバレッジ達成**: 3,490 剤全て全成分が分類済み（コード or ステータス）。

### 他セッション向け引き継ぎメモ (v0.1.1 から継承 + v0.2.0 追記)
- **CP932 制約**: `build_bundled.py` `verify_data.py` `update_data.py` の
  コンソール出力に `✓` `✗` `→` 等の非 ASCII 記号を入れない。
  入れる場合は `print` 前に `sys.stdout.reconfigure(encoding="utf-8")` を呼ぶ。
  （docstring/コメント内の `→` は print されないので問題なし）
- **bash から手動で Python を呼ぶ場合**は環境変数 `PYTHONIOENCODING=utf-8` が必須。
  `データ更新.bat` 経由なら `set` 済み。
- **失効剤を取り込むには `xlrd==1.2.0`**（2.x は `.xls` 非対応）。
  未インストールだと `build_data.py` が黙ってスキップし、`pesticides.json` が
  約 1.6 MB（現行剤のみ）に縮む。インストール済みなら 4.7 MB（11,267 件）。
- **`generated_at`** は `build_data.py` が `pesticides.json` に書く ISO 日付。
  HTML ヘッダ表示はこの値だけが情報源 — `build_data.py` を実行しない限り
  HTML を再ビルドしても表示は変わらない。
- **FAMIC CSV ファイル名は月ごとに変わる** (`R0804080.csv` の部分)。
  v0.2.0 から `scripts/build_data.py` が `R*0.csv` / `R*1.csv` / `R*2.csv` を
  glob 検出するので、新 CSV を `rawdata/famic/` に置けば自動で認識する。
  旧 CSV は `scripts/update_data.py` が自動削除する。
- **`rawdata/rac_manual.json`**: RAC 未分類成分の手動ラベル。
  ステータスは `out_of_scope` / `legacy` / `recent` の3択。
  新しい成分を追加/既存を修正したら `build_data.py` 再実行で反映。
  RAC コードは手動付与しない (CropLife 公式分類のみを信頼)。
- **リリース手順は Skill 化済み**: `~/.claude/skills/pesticide-search-release/SKILL.md`。
  「バージョンアップ」「リリース」「/pesticide-search-release」で起動。
  VERSION / `index.html` のバージョンタグ / CHANGELOG / README / バンドル HTML を
  まとめて更新する。

## v0.1.0 - 2026-04-22

### 初回リリース
- FAMIC 公開 CSV（基本部＋適用部）と CropLife Japan RAC コード Excel を統合した
  オフライン検索ツール。
- 商品名・成分名・RAC コード（IRAC/FRAC/HRAC）・登録番号の双方向検索。
- 用途フィルタ（殺虫剤/殺菌剤/除草剤）、家庭向け自動除外、混合剤絞り込み。
- 適用部（対象作物・病害虫・使用量・使用時期）の詳細表示。
- TSV / CSV / Markdown / JSON エクスポート。
- シングル HTML バンドル配布（ダブルクリック起動、完全オフライン動作）。
- CSP で外部通信を禁止し、データ漏洩ゼロを保証。

### データ統計
- 商品総数: 3,490 件 (殺虫剤 1,243 / 殺菌剤 799 / 除草剤 1,448)
- RAC コードマッチ率: 98.3%
- 適用部エントリ: 96,720 件
- 混合剤: 45.3%
- 家庭向け除外: 42 件

### 既知の制約
- FAMIC 公開 CSV は現行登録剤のみ含み、**失効剤リストは含まれません**。
  「失効剤色分け表示」は将来バージョン（要外部データ源）で対応予定。
- JPP-NET は会員制フォーム認証・公開 API 無し・CORS 不対応のため、連携を断念。
- RAC 未分類の成分が 51 種類（主に新規登録成分）。手動マッピングで後日対応予定。
