# Changelog

## Ver. 1.4.0 (v1.4.0) - 2026-07-10

### 新機能
- **検索モードに AND / OR トグルを追加**: 検索バー右横のボタンで、複数語を
  「すべて含む (AND, 既定)」/「いずれか含む (OR)」に切り替えられるようにした。
  OR は各語の検索結果の和集合を返す。

### 改善 (検索精度)
- **AND 検索で「作物 × 病害虫」を同一適用行での共起に限定**: 従来は「ぶどう
  うどんこ病」で、ぶどう(すす点病のみ)と別作物のうどんこ病を別々の適用行で
  拾って一致させていた。作物トークンと病害虫トークンのペアが同じ適用行に
  登録されている商品だけを返すよう修正 (例: 硫黄粉剤系のようにぶどうへは
  すす点病のみの商品を除外)。
  - トークンの役割 (作物/病害虫) はデータベース全体の語彙で判定し、"なす" が
    病害虫名 "ヨメナスジハモグリバエ" に部分一致しても作物として扱う。
  - 作物×作物 / 病害虫×病害虫 のように片側のみの組み合わせは従来どおり独立
    AND (別々の適用行でも可) を維持。作物×複数病害虫も各ペアが成立すれば一致。
- **サイドバーの「作物」「病害虫」同時指定も同一適用行での共起に統一**。

## Ver. 1.3.0 (v1.3.0) - 2026-06-18

### 新機能
- **RACコード検索をIRAC/FRAC/HRAC系統別に絞り込めるラジオボタンを追加**:
  数字単独のRACコード(例: "3")がIRAC/FRAC/HRACで意味が異なるのに同一視され
  検索結果に混ざる問題に対応。検索インデックスを系統別(I/F/H/all)に分け、
  サイドバーのラジオボタンで検索対象系統を選べるようにした。検索ボックスで
  "IRAC 4A" 等と明示入力した場合はラジオボタンの設定より優先される。

  カテゴリ不明として除外されていた問題を修正

  - 動的成分ヒント(現行剤由来)に加え、上位頻度の歴史的化合物名約95件をWebSearchで
    個別調査・確認した上でLEGACY_CATEGORY_HINTSに追加
    (殺ダニ専用剤は現行データの実例に基づき「殺虫剤」に分類)
  - 確信を持てなかった項目は捏造防止のため追加せず除外を維持
  - 失効剤: 7,778 → 18,618件 / カテゴリ不明での除外: 5,799 → 2,629件

## Ver. 1.2.1 (v1.2.1) - 2026-05-06

### 分類精度の修正
- **石灰硫黄合剤・水和硫黄剤・松脂合剤を「殺虫殺菌剤」に正しく分類**:
  これらは硫黄系・松脂系で殺ダニ・カイガラムシ防除を主目的とし、殺虫殺菌両作用を持つ
  剤として登録されているが、FAMIC データ上は用途欄が「石灰硫黄合剤」「殺菌剤」等の
  独自/単一表記となっており、これまで殺菌剤としてのみ分類されていた。
  - 影響商品数: 石灰硫黄合剤 176 件 + 水和硫黄剤 116 件 + 松脂合剤 4 件 = 計 296 件
  - 殺虫剤カテゴリの総数が 5,663 → 5,957 件 (+294 件) に増加
- **`scripts/build_data.py` に `TYPE_NAME_CATEGORY_OVERRIDE` を導入**:
  type_name (基本部の「種類」欄) を見て、FAMIC 表記と実際の作用機序が乖離する剤を
  上書き分類するマップ。今後同様の誤分類が見つかった場合はここに追加するだけで対応可能。
- **グループカードのカテゴリ表示を「全商品の和集合」に変更**:
  これまで代表商品のカテゴリのみを表示していたため、混在グループ (クロルピクリンくん蒸剤・
  BT粒剤・醸造酢液剤・PCP剤など、計 5 種類) で一部のカテゴリが見落とされていた。
  グループ内のすべての商品のカテゴリを和集合で表示することで、混在剤も正確に把握可能に。

### 他セッション向け引き継ぎメモ (v0.1.1 から継承 + v0.2.0 追記)
- **CP932 制約**: `build_bundled.py` `verify_data.py` `update_data.py` の
  コンソール出力に `✓` `✗` `→` 等の非 ASCII 記号を入れない。
  入れる場合は `print` 前に `sys.stdout.reconfigure(encoding="utf-8")` を呼ぶ。
  （docstring/コメント内の `→` は print されないので問題なし）
- **bash から手動で Python を呼ぶ場合**は環境変数 `PYTHONIOENCODING=utf-8` が必須。
  `データ更新.bat` 経由なら `set` 済み。
- **失効剤を取り込むには `xlrd==1.2.0`**（2.x は `.xls` 非対応）。
  未インストールだと `build_data.py` が黙ってスキップし、`pesticides.json` が
  約 1.6 MB（現行剤のみ）に縮む。インストール済みなら 4.7 MB（11,267 件）。
- **`generated_at`** は `build_data.py` が `pesticides.json` に書く ISO 日付。
  HTML ヘッダ表示はこの値だけが情報源 — `build_data.py` を実行しない限り
  HTML を再ビルドしても表示は変わらない。
- **FAMIC CSV ファイル名は月ごとに変わる** (`R0804080.csv` の部分)。
  v0.2.0 から `scripts/build_data.py` が `R*0.csv` / `R*1.csv` / `R*2.csv` を
  glob 検出するので、新 CSV を `rawdata/famic/` に置けば自動で認識する。
  旧 CSV は `scripts/update_data.py` が自動削除する。
- **`rawdata/rac_manual.json`**: RAC 未分類成分の手動ラベル。
  ステータスは `out_of_scope` / `legacy` / `recent` の3択。
  新しい成分を追加/既存を修正したら `build_data.py` 再実行で反映。
  RAC コードは手動付与しない (CropLife 公式分類のみを信頼)。
- **リリース手順は Skill 化済み**: `~/.claude/skills/pesticide-search-release/SKILL.md`。
  「バージョンアップ」「リリース」「/pesticide-search-release」で起動。
