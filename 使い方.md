# browser-pesticide-search

農薬登録情報のオフライン検索ツール。FAMIC 公開 CSV と CropLife Japan の
RAC コード表を統合し、**商品名 ↔ 成分名 ↔ IRAC/FRAC/HRAC コード** を
双方向に検索できるシングル HTML ツールです。

## 特徴

- **オフライン完結**: ダブルクリックで起動、ネットワーク不要
- **双方向検索**: 商品名でも成分名でも RAC コードでも引ける
- **耐性管理対応**: IRAC（殺虫剤）/FRAC（殺菌剤）/HRAC（除草剤）コードで
  同じ作用機作の剤をまとめて絞り込み
- **混合剤対応**: 1剤に複数成分を持つ混合剤も正しく展開
- **詳細情報**: クリックで対象作物・病害虫・希釈倍数・使用時期まで表示
- **エクスポート**: TSV/CSV/Markdown/JSON で選択結果をダウンロード
- **プライバシー**: CSP で外部通信を禁止。データが外部に出ることはありません
- **最終更新日表示**: ヘッダに「最終更新: 令和X年X月X日」を常時表示。
  `データ更新.bat` / `.command` を実行するたびに自動的に反映されます

## 使い方

### 配布版（推奨）

| OS | 手順 |
|----|------|
| Windows | エクスプローラで `build\pesticide_search_bundled.html` をダブルクリック |
| macOS | Finder で `build/pesticide_search_bundled.html` をダブルクリック |

既定のブラウザで開きます（約 25 MB、初回読込 1〜3 秒）。
ネットワーク接続は不要、データはすべて HTML 内に埋め込み済みです。

> **キャッシュ注意**: データ更新後に開く際はブラウザで強制再読み込みしてください。
> Windows: `Ctrl + F5` / macOS: `Cmd + Shift + R`

### 開発版

ローカル HTTP サーバを立てて `src/index.html` を直接開きます。
ファイル直開き (`file://`) では `fetch()` がブロックされるため必ず HTTP 経由で。

| OS | コマンド |
|----|---------|
| Windows | `cd src && py -3 -m http.server 8765` |
| macOS   | `cd src && python3 -m http.server 8765` |

その後ブラウザで http://localhost:8765/ を開く。停止は `Ctrl + C`。

### 検索例
| 入力 | 結果 |
|------|------|
| `イミダクロプリド` | 成分「イミダクロプリド」を含む全商品 |
| `アドマイヤー` | 商品名に「アドマイヤー」を含む剤 |
| `4A` or `IRAC 4A` | IRAC コード 4A（ネオニコチノイド系）の全商品 |
| `M7` | FRAC コード M7 の全殺菌剤 |
| `第12345号` or `12345` | 登録番号で直接引く |
| `バイエル` | 会社名で絞り込み |

**RAC コード検索の系統指定 (Ver. 1.3 から)**: 数字単独のコード（例: `3`）は
IRAC/FRAC/HRAC で意味が異なるため、サイドバーのラジオボタンで検索対象の系統
（IRAC / FRAC / HRAC / すべて）を選べます。検索ボックスに `IRAC 4A` のように
系統を明示入力した場合は、ラジオボタンの設定より優先されます。

## データ更新

データは月初 FAMIC 更新があります。

### 推奨: ワンクリック更新（ダブルクリックで起動）

| OS | ファイル |
|----|---------|
| macOS | `データ更新.command` |
| Windows | `データ更新.bat` |

FAMIC index ページから最新 zip URL を自動抽出して取得→展開→JSON生成→
シングルHTML再ビルド、を 1〜2 分で実行します。CropLife RAC 表も同時取得。

**前提**: Python 3.8 以上。
- macOS は標準の `python3`（無ければ `xcode-select --install`）
- Windows は [python.org](https://www.python.org/downloads/windows/) か
  Microsoft Store で "Python 3" を入れ、インストール時に
  "Add Python to PATH" にチェック。`openpyxl` は初回自動インストール。

### 手動更新（コマンドライン派向け）

```bash
python3 scripts/update_data.py
```

上記 `.command` / `.bat` はこのスクリプトを呼んでいるだけです。

## データソース

- [FAMIC 農薬登録情報ダウンロード](https://www.acis.famic.go.jp/ddata/index2.htm)
  （公開 CSV、認証不要、月初更新）
- [CropLife Japan RAC コード検索表](https://www.croplifejapan.org/labo/mechanism.html)
  （Excel、無償公開、2〜3ヶ月ごと更新）

## 制約

- **JPP-NET 連携なし**。JPP-NET は会員制フォーム認証で公開 API が無く、
  ブラウザ単体 HTML からの自動参照は技術的・規約的に不可のため断念しました。
- **RAC 分類カバレッジ 100% 達成** (v0.2.0): 成分名フォールバック + 手動ラベル
  (`rawdata/rac_manual.json`) で全 3,490 剤の全成分に分類（コード or ステータス）を
  付与。RAC 対象外の物理剤/生物製剤/軟体動物剤/無機剤、古典剤、新規登録剤には
  `対象外` / `古典未分類` / `新規未収録` のバッジを表示。
- **失効剤取込には `xlrd==1.2.0` が必要**。xlrd 2.x 以降は `.xls` 非対応のため
  `rawdata/sikkou/*.xls` が読めず、`pesticides.json` が現行剤のみ (~1.6 MB) に
  縮小します。正しく取り込めば 11,267 件 (現行 3,490 + 失効 7,777) / ~4.7 MB。
